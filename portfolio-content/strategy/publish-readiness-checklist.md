# Publish Readiness Checklist

Use this checklist before turning the Markdown content into the public portfolio website.

## Client and Confidentiality Approval

- Confirm which client names can be shown publicly.
- Confirm whether HSBC, Invesco, Carlyle, EBRD, Novartis, and other names can be used.
- If not approved, replace with neutral labels such as "global banking client", "investment management client", or "healthcare enterprise client".
- Confirm whether Appian Accelerate advisory details can be published.
- Confirm whether screenshots contain client-sensitive data.

## Ownership Confirmation

For every project, confirm:

- authored
- co-authored
- implemented
- reviewed
- advised
- supported

Do not imply sole ownership when work was collaborative.

## Screenshot and Asset Approval

Before publishing visuals:

- remove client data
- remove user names
- remove email/phone values
- remove tokens/API keys
- remove internal object IDs where unnecessary
- blur or replace confidential business data
- confirm permission to show UI screenshots

## Metrics Confirmation

Only publish metrics after validation.

Metrics to confirm:

- ClaimRoute IQ extraction accuracy
- claim processing time reduction
- reviewer workload reduction
- KCC answer accuracy
- KCC context/token reduction
- KCC evidence highlight accuracy
- Appian Accelerate performance improvements
- Excel export runtime improvements
- reconciliation volume
- revenue-cycle improvement
- user experience/load-time improvement

## Plugin Confirmation

For every plugin, confirm:

- exact plugin name
- plugin type
- version
- implementation ownership
- supported Appian versions
- supported browsers
- accessibility behavior
- known limitations
- documentation/demo links

## Needs Confirmation Pages

These pages should remain draft-only until evidence is added:

- Locale-Based Date and DateTime Picker Plugin
- HTML/DOCx to Accessible PDF Plugin
- GDS UK Passport Service POC

## Research Paper Citation

Before publishing the research paper section:

- add publication URL if available
- add DOI if available
- add volume/issue/page details if available
- confirm exact title capitalization

## Final Website Content QA

- Check all project categories.
- Check all evidence labels.
- Remove internal typos and raw Appian object names from public copy.
- Confirm all links work.
- Confirm resume download is current.
- Confirm contact details are correct.
- Confirm mobile readability.
- Confirm accessibility contrast.
- Confirm no secret-like values appear in content or screenshots.
