# Content Checklist

## Source Order

1. `portfolio-content/profile/`
2. `portfolio-content/projects/`
3. `portfolio-content/strategy/`
4. `portfolio-content/source-copies/source-materials-used.md`
5. `development-ready/content-data-architecture.md`

## Public-Safe Checks

- No raw client-sensitive data.
- No private URLs.
- No environment names.
- No server names.
- No Appian object IDs.
- No raw logs.
- No private key content.
- No unverified numbers.
- No unsupported ownership claims.

## Portfolio Copy Pattern

Use this shape:

- problem
- context
- role
- architecture or solution approach
- UX/product decisions
- business value
- technologies
- evidence status

## Category Rules

- Client Projects: delivery or advisory engagements.
- Plugins: Appian component or smart service plugin work.
- Innovative Projects: POCs, AI products, product-style demos.
- Accelerators: reusable patterns, advisory assets, diagnostics, governance or implementation patterns.
