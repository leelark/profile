# SDLC Checklist

## Readiness

- Read `development-ready/README.md`.
- Confirm the task is content, design, architecture, implementation, QA, release, or maintenance.
- Select the relevant project skill.
- Confirm no implementation is requested when the user asks for planning only.

## Planning

- Use `development-ready/technical-solution-design.md`.
- Use `development-ready/functional-requirements.md`.
- Use `development-ready/content-data-architecture.md`.
- Confirm GitHub Pages hosting mode: user site, project site, or custom domain.
- Preserve dark default and light theme option.
- Preserve Splitforms contact requirement.

## Implementation Handoff

- Use `development-ready/sdlc-roadmap-and-backlog.md`.
- Break work into phases.
- Define acceptance criteria.
- Identify verification commands before coding starts.

## QA

- Use `development-ready/qa-accessibility-performance-seo.md`.
- Check accessibility, responsiveness, SEO, performance, visual polish, content integrity, and contact form.

## Release

- Use `development-ready/release-and-operations-plan.md`.
- Confirm domain, form, analytics, monitoring, launch checklist, and rollback plan.
