# Portfolio SDLC Orchestrator

## Core Rule

Keep the portfolio goal central: build a powerful, premium, modern, responsive, animated, sleek, public-safe portfolio website that fully represents Leelark's profile, experience, projects, plugins, accelerators, Appian expertise, AI work, architecture capability, and enterprise solution design.

Do not scaffold, implement, install packages, or create website source code unless the user explicitly asks for development. When implementation is requested, first read the relevant development-ready specification and select the right project skill.

## First Files To Read

From repository root:

1. `context.md`
2. `AGENTS.md`
3. `development-ready/README.md`

## Skill Routing

Use:

- `portfolio-content-system` for content, case studies, evidence, copy, categories, public-safe review.
- `portfolio-design-system` for visual direction, theme, layout, animation, responsive UX.
- `portfolio-frontend-architecture` for stack, routing, data model, component architecture.
- `portfolio-frontend-implementation` for actual frontend build work after the user explicitly requests implementation.
- `portfolio-implementation-planner` for backlog, tasks, sequencing, acceptance criteria.
- `portfolio-contact-form` for Splitforms contact form work.
- `portfolio-qa-accessibility` for accessibility, performance, SEO, responsive, content-integrity QA.
- `portfolio-release-operations` for deployment, launch, rollback, monitoring, maintenance.

## SDLC Flow

Follow this order unless the user explicitly narrows the task:

1. Confirm scope and current repo state.
2. Read the relevant specs.
3. Identify affected content/design/architecture areas.
4. Make the smallest coherent update.
5. Validate links, structure, and consistency.
6. Run or define QA appropriate to the phase.
7. Update docs and indexes when new artifacts are added.

## Public-Safe Guardrail

Never publish or copy raw secrets, raw logs, Appian object IDs, environment names, server names, private URLs, private key files, raw exports, unapproved client names, or unverified metrics.
