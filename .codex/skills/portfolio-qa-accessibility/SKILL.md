# Portfolio QA and Accessibility

## Core Rule

Verify before claiming complete. Check accessibility, responsiveness, performance, SEO, visual quality, content integrity, public safety, contact form behavior, and reduced motion.

## First Files To Read

From repository root:

1. `development-ready/qa-accessibility-performance-seo.md`
2. `development-ready/functional-requirements.md`
3. `development-ready/contact-form-specification.md`
4. `portfolio-content/strategy/publish-readiness-checklist.md`

## QA Categories

Check:

- accessibility
- responsive layout
- performance
- SEO metadata
- visual polish
- content safety
- contact form
- reduced motion
- internal links
- category consistency

## Accessibility Targets

Target WCAG 2.2 AA.

Prioritize:

- keyboard navigation
- focus visibility
- contrast
- form labels
- semantic headings
- reduced motion
- no color-only status

## Content Integrity

Check for:

- raw client data
- internal names
- Appian object IDs
- private URLs
- unverified metrics
- missing evidence status
- incomplete project sections
