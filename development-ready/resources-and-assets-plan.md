## Design Asset Rules

Use visuals that explain real work:

- screenshots
- sanitized mockups
- diagrams
- workflow visuals
- product UI snapshots

Avoid:

- generic stock images
- decorative blobs
- dark blurred hero backgrounds
- purely atmospheric AI images
- screenshots with client-sensitive data

## Diagram Style

Use:

- dark/light theme compatible colors
- neutral surfaces
- purple for AI/plugin emphasis
- blue for integration/data flow
- green for validation/outcomes
- amber for caution/needs confirmation

Diagram types:

- system architecture
- workflow sequence
- decision flow
- before/after UX
- data pipeline
- plugin boundary model

## Asset Sanitization

Before public use:

- blur or replace client names
- remove user names
- remove environment names
- remove URLs
- remove IDs
- remove internal comments
- remove exact metrics unless approved

## Placeholder Strategy

If screenshots are not available:

- use abstract but specific system diagrams
- use sanitized wireframes
- use content-first cards
- use architecture visualizations

Do not use placeholder stock images that make the portfolio feel generic.
