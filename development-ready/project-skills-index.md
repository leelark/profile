# Project Skills Index

## Purpose

This file lists the project-local Codex skills created for the portfolio website SDLC.

Skill location:

- `.codex/skills/`

## Skills

### portfolio-sdlc-orchestrator

Use for:

- coordinating development sessions
- choosing which project skill to use
- preserving SDLC order
- keeping work aligned to content, design, QA, and release requirements

### portfolio-content-system

Use for:

- project content
- case-study copy
- profile/resume content
- category organization
- evidence status
- public-safe review

### portfolio-design-system

Use for:

- visual theme
- dark/light mode
- responsive UX
- motion and animation direction
- page layout polish
- premium UI review

### portfolio-frontend-architecture

Use for:

- stack decisions
- routing
- component architecture
- data model
- performance architecture
- security and privacy planning

### portfolio-frontend-implementation

Use for:

- actual frontend implementation after the user explicitly requests development
- Astro scaffold and route creation
- Tailwind theme implementation
- React components
- Motion interactions
- content data rendering
- contact form implementation

### portfolio-implementation-planner

Use for:

- turning specs into development tasks
- backlog sequencing
- acceptance criteria
- milestone planning
- implementation handoff

### portfolio-contact-form

Use for:

- Splitforms contact form
- form accessibility
- validation
- thank-you redirect
- domain lock and spam setup requirements

### portfolio-qa-accessibility

Use for:

- accessibility review
- responsive QA
- performance QA
- SEO QA
- content integrity checks
- visual polish review

### portfolio-release-operations

Use for:

- deployment planning
- environment setup
- launch checklist
- rollback planning
- monitoring and maintenance
